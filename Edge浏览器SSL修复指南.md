# Edge浏览器SSL显示"常规"修复指南

## 问题说明

Edge显示"常规(G)"而不是"已验证"或"安全"，虽然证书是正确的Let's Encrypt证书。这通常是Edge的HSTS缓存问题。

## 解决方案

### 方法1：清除Edge的HSTS缓存

1. **打开Edge的HSTS管理页面**
   - 在Edge地址栏输入：
   ```
   edge://net-internals/#hsts
   ```

2. **删除域名的HSTS策略**
   - 在 "Delete domain security policies" 部分
   - 输入：`hello1984.net`
   - 点击 "Delete" 按钮

3. **清除Edge缓存**
   - 按 `Ctrl + Shift + Delete`
   - 选择"全部时间"
   - 勾选以下选项：
     - ✅ 缓存的图像和文件
     - ✅ Cookie和其他网站数据
   - 点击"立即清除"

4. **完全关闭Edge**
   - 确保所有Edge窗口都关闭
   - 检查任务管理器，确保没有Edge进程在运行

5. **重新打开Edge并访问**
   - 打开Edge
   - 访问 `https://hello1984.net`
   - 应该显示"已验证"或"安全"

### 方法2：使用Edge的隐私模式测试

1. **打开InPrivate窗口**
   - 按 `Ctrl + Shift + N`
   - 访问 `https://hello1984.net`

2. **如果隐私模式正常**
   - 说明是缓存问题
   - 按照方法1清除缓存

### 方法3：重置Edge的SSL状态

1. **打开Edge设置**
   - 点击右上角三个点 → 设置

2. **清除浏览数据**
   - 隐私、搜索和服务 → 清除浏览数据
   - 选择"立即清除"
   - 勾选"缓存的图像和文件"和"Cookie和其他网站数据"

3. **重启Edge**

### 方法4：检查Windows证书存储

1. **打开证书管理器**
   - 按 `Win + R`
   - 输入：`certmgr.msc`
   - 回车

2. **检查不受信任的证书**
   - 展开"不受信任的证书" → "证书"
   - 查找是否有 `hello1984.net` 或 `38.175.195.104` 的证书
   - 如果有，右键删除

3. **检查中间证书**
   - 展开"中间证书颁发机构" → "证书"
   - 查找 "Let's Encrypt" 相关证书
   - 确保它们存在且有效

### 方法5：更新Edge浏览器

1. **检查Edge版本**
   - 地址栏输入：`edge://settings/help`
   - 查看版本号

2. **更新到最新版本**
   - Edge会自动更新
   - 或者手动检查更新

## 验证步骤

清除缓存后：

1. **访问网站**
   - 打开Edge（或InPrivate窗口）
   - 访问 `https://hello1984.net`

2. **检查证书状态**
   - 点击地址栏的锁图标
   - 应该显示：
     - ✅ "已验证" 或 "安全"
     - ✅ 证书由 Let's Encrypt 颁发
     - ✅ 连接是安全的

3. **查看证书详情**
   - 点击锁图标 → 连接是安全的 → 证书有效
   - 应该显示完整的证书信息

## Edge显示级别的说明

- **已验证**：证书完全有效，由受信任的CA颁发
- **常规(G)**：证书有效，但可能因为缓存或其他原因未显示为"已验证"
- **不安全**：证书无效或过期

## 如果问题仍然存在

1. **等待几分钟**
   - Edge的缓存可能需要时间更新

2. **检查系统时间**
   - 确保Windows系统时间正确
   - 错误的系统时间会导致证书验证问题

3. **检查Windows更新**
   - 确保Windows和Edge都是最新版本

4. **使用其他浏览器对比**
   - 如果Chrome正常，说明服务器配置正确
   - Edge的问题应该在客户端

## 服务器端状态

✅ 证书：Let's Encrypt（正确）
✅ SSL Labs评级：A+
✅ Chrome：正常显示
✅ 证书链：完整

问题应该在Edge的缓存，清除HSTS缓存后应该可以解决。
