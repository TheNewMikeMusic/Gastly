# 清除浏览器SSL缓存 - 解决"不安全"提示

## 问题说明

SSL Labs显示A+评级，证书配置正确，但浏览器仍显示"不安全"。这通常是浏览器缓存了旧的SSL状态（特别是HSTS）。

## 解决方案

### 方法1：清除Chrome的HSTS缓存（推荐）

1. **打开Chrome设置**
   - 地址栏输入：`chrome://net-internals/#hsts`

2. **删除域名的HSTS策略**
   - 在 "Delete domain security policies" 部分
   - 输入：`hello1984.net`
   - 点击 "Delete" 按钮

3. **清除浏览器缓存**
   - 按 `Ctrl + Shift + Delete`
   - 选择"全部时间"
   - 勾选"缓存的图片和文件"
   - 点击"清除数据"

4. **重启浏览器**
   - 完全关闭Chrome（确保所有窗口关闭）
   - 重新打开Chrome

5. **使用隐私模式测试**
   - 按 `Ctrl + Shift + N` 打开隐私窗口
   - 访问 `https://hello1984.net`

### 方法2：清除Edge的HSTS缓存

1. **打开Edge设置**
   - 地址栏输入：`edge://net-internals/#hsts`

2. **删除域名的HSTS策略**
   - 在 "Delete domain security policies" 部分
   - 输入：`hello1984.net`
   - 点击 "Delete" 按钮

3. **清除浏览器数据**
   - 按 `Ctrl + Shift + Delete`
   - 选择"全部时间"
   - 清除缓存

### 方法3：重置Chrome的SSL状态

1. **完全重置Chrome**
   - 关闭所有Chrome窗口
   - 删除Chrome的用户数据目录（会清除所有数据）
   - 或者使用隐私模式

2. **使用隐私模式（最简单）**
   - 按 `Ctrl + Shift + N`
   - 访问 `https://hello1984.net`
   - 如果隐私模式正常，说明是缓存问题

### 方法4：清除Windows证书存储（如果使用Windows）

1. **打开证书管理器**
   - 按 `Win + R`
   - 输入：`certmgr.msc`
   - 回车

2. **检查不受信任的证书**
   - 展开"不受信任的证书" → "证书"
   - 查找是否有 `hello1984.net` 或 `38.175.195.104` 的证书
   - 如果有，删除它们

3. **检查中间证书**
   - 展开"中间证书颁发机构" → "证书"
   - 检查是否有问题

### 方法5：使用其他浏览器测试

1. **Firefox**
   - 按 `Ctrl + Shift + Delete`
   - 清除"缓存"和"站点数据"
   - 访问 `https://hello1984.net`

2. **Safari（Mac）**
   - 菜单 → 清除历史记录
   - 选择"全部历史记录"

## 验证步骤

清除缓存后：

1. **访问网站**
   - 打开隐私模式
   - 访问 `https://hello1984.net`

2. **检查证书**
   - 点击地址栏的锁图标
   - 查看证书信息
   - 应该显示：
     - ✅ 连接是安全的
     - ✅ 证书由 Let's Encrypt 颁发
     - ✅ 证书有效

3. **检查控制台**
   - 按 `F12` 打开开发者工具
   - 查看 Console 标签
   - 不应该有SSL/证书错误

## 如果问题仍然存在

1. **检查系统时间**
   - 确保系统时间正确
   - 错误的系统时间会导致证书验证失败

2. **检查防火墙/安全软件**
   - 某些安全软件可能拦截SSL连接
   - 临时禁用测试

3. **检查DNS**
   - 确保DNS解析正确
   - `nslookup hello1984.net` 应该返回 `38.175.195.104`

4. **联系我**
   - 如果以上方法都不行，告诉我具体的错误信息
